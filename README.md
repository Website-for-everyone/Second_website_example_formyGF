## 🚀 Как быстро и беспроблемно выгрузить сайт на GitHub из Google AI Studio

Мы подготовили и оптимизировали всё необходимое для мгновенного запуска. Проект использует полностью относительные пути для ресурсов (изображения, аудио) и снабжён готовым скриптом авто-деплоя.

Сделайте следующие простые шаги:

### Шаг 1: Экспорт из Google AI Studio
1. В правом верхнем углу интерфейса Google AI Studio нажмите на иконку настроек / экспорта.
2. Выберите **Export to GitHub** (или скачайте ZIP-архив и загрузите его в ваш новый GitHub репозиторий).

### Шаг 2: Настройка доступа для автодеплоя на GitHub
Благодаря созданному нами файлу `.github/workflows/deploy.yml`, GitHub автоматически скомпилирует и опубликует ваш сайт при каждой вашей загрузке. Нужно лишь один раз дать Action права на запись:
1. Перейдите в ваш репозиторий на GitHub.
2. Откройте вкладку **Settings** (Настройки репозитория) -> **Actions** -> **General**.
3. Прокрутите страницу вниз до раздела **Workflow permissions** (Права воркфлоу) и выберите **Read and write permissions** (Разрешить чтение и запись).
4. Нажмите **Save** (Сохранить).

### Шаг 3: Включение хостинга GitHub Pages
1. Перейдите во вкладку **Settings** репозитория.
2. Слева найдите раздел **Pages**.
3. В выпадающем меню **Source** выберите **Deploy from a branch** (Развертывание из ветки).
4. В разделе **Branch** выберите ветку `gh-pages` (она автоматически создастся после того, как воркфлоу завершит первую сборку) и папку `/ (root)`.
5. Нажмите **Save**.

🎉 **Всё готово!** Через минуту ваш шикарный интерактивный сайт будет доступен по адресу `https://<ваш-логин-на-github>.github.io/<имя-репозитория>/`.

## 🛠️ Запуск проекта на локальном компьютере

Если вы хотите запустить проект локально:

1. Установите зависимости:
   ```bash
   npm install
   ```
2. Запустите сервер разработки:
   ```bash
   npm run dev
   ```
3. Откройте в браузере `http://localhost:3000` (или порт, указанный в CLI).

## ✨ Что было оптимизировано для работы на GitHub Pages:
* **Относительные ссылки (`base: './'`)**: Наш конфигурационный файл `vite.config.ts` теперь использует относительный базовый путь. Это избавляет от ошибок 404, если репозиторий находится во вложенных папках страниц GitHub.
* **Стриминг аудио и ассеты**: Все изображения и аудиозапись `track.mp3` теперь компилируются Vite как модульные статические ассеты. Они никогда не пропадут и не вызовут ошибку загрузки.
* **GitHub Actions Workflow**: Полная автоматизация. Вам больше не нужно собирать проект вручную перед деплоем.
